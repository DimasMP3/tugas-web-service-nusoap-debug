Nama : [Isi Nama Anda/Teman 1] 		Angka hasil lemparan balikan web service (response node) selalu kosong di browser client meskipun tidak memunculkan indikasi error.
[GAMBAR: Output bertuliskan "Hasil penjumlahan 10 dan 25 adalah " tanpa angka akhir perhitungan/bernilai null/0]
Pada badan function jumlahkan($a, $b) server, metode dibuat tidak mereturn variabel $hasil secara eksplisit ke memori program.

Sebelum:
function jumlahkan($a, $b) {
    $hasil = $a + $b;
    // Lupa memberikan perintah lemparan nilai keluar
}
Sesudah:
function jumlahkan($a, $b) {
    $hasil = $a + $b;
    return $hasil;
}
Kini Payload XML meresponse tipe data integer dan angka muncul utuh di *echo* statement.
[GAMBAR: Output string lengkap: Hasil penjumlahan 10 dan 25 adalah 35]
Nim : [Isi NIM]
Nama : [Isi Nama Teman 2] 		Peringatan error dari SOAP client karena format output balikan tidak diketahui format meta data tipe pengembalian nilainya.
[GAMBAR: Notice/Error return map atau Client gagal mengenali service namespace dari register]
Saat melakukan eksekusi $server->register() di script server.php, informasi array keluaran dan spesifikasi format payload RPC masih terbelengkalai belum disebutkan.

Sebelum:
$server->register('kurangi',
    array('x' => 'xsd:int', 'y' => 'xsd:int')
);
Sesudah:
$server->register('kurangi',
    array('x' => 'xsd:int', 'y' => 'xsd:int'),
    array('return' => 'xsd:int'),
    $namespace,
    false,
    'rpc',
    'encoded',
    'Mengurangi dua bilangan'
);
Struktur tipe balikan return SOAP terdata valid sehingga operasi bisa diurai oleh obyek dari bahasa pemrograman apapun.
